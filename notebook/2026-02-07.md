# 2026-02-07

## Objective
Refine the logic for distinguishing between "dirty panels" and "cloudy conditions."

## Record of what was done
- Discussed the limitations of relying purely on the MPPT power data. If a cloud passes over the array, the power will drop by >25%, which could trigger a false cleaning cycle.

## Decision
- Integrate ambient light sensors into the system. The ESP32 will cross-reference the panel's power output against the ambient light sensor data.
- If ambient light is high but power is low, the panel is dirty.
- If both are low, the array is simply shaded or under cloud cover.

## Notes
- Considered a per-panel ambient light sensor vs. a single global one. Decided a global sensor is sufficient since shading is usually broad and we can supplement with the per-panel voltage comparison to localize.
