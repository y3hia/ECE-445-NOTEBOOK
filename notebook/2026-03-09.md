# 2026-03-09

## Objective
Develop LEDC PWM generation and verify the breadboard buck converter.

## Record of what was done
- Borrowed an ESP32-C3 Super Mini development board to use for our breadboard demo while we wait for the final ESP32-S3 chips.
- Wrote the C code to configure the ESP32's hardware LEDC peripheral to generate the high-frequency PWM needed to switch the MOSFETs on the buck converter.
- Set the frequency to 5 kHz for initial validation.
- Connected the ESP32-C3 to the board and verified that the buck converter successfully regulates voltage under load.

## Code Snippet
```c
#include <stdio.h>
#include "driver/ledc.h"
#include "esp_err.h"

#define LEDC_TIMER              LEDC_TIMER_0
#define LEDC_MODE               LEDC_LOW_SPEED_MODE
#define LEDC_OUTPUT_IO          (8)
#define LEDC_CHANNEL            LEDC_CHANNEL_0
#define LEDC_DUTY_RES           LEDC_TIMER_13_BIT
#define LEDC_FREQUENCY          (5000)

void app_main(void)
{
    ledc_timer_config_t ledc_timer = {
        .speed_mode       = LEDC_MODE,
        .timer_num        = LEDC_TIMER,
        .duty_resolution  = LEDC_DUTY_RES,
        .freq_hz          = LEDC_FREQUENCY,
        .clk_cfg          = LEDC_AUTO_CLK
    };
    ESP_ERROR_CHECK(ledc_timer_config(&ledc_timer));

    ledc_channel_config_t ledc_channel = {
        .speed_mode     = LEDC_MODE,
        .channel        = LEDC_CHANNEL,
        .timer_sel      = LEDC_TIMER,
        .intr_type      = LEDC_INTR_DISABLE,
        .gpio_num       = LEDC_OUTPUT_IO,
        .duty           = 0,
        .hpoint         = 0
    };
    ESP_ERROR_CHECK(ledc_channel_config(&ledc_channel));

    uint32_t target_duty = 4096;
    while(1) {
        ledc_set_duty(LEDC_MODE, LEDC_CHANNEL, target_duty);
        ledc_update_duty(LEDC_MODE, LEDC_CHANNEL);
        vTaskDelay(pdMS_TO_TICKS(1000));
    }
}
```

## Notes
- 5 kHz is far below the eventual 100 kHz target, but it's enough to validate the basic LEDC config and gate driver chain.
- Will switch to Arduino-IDE-style code once integration starts; ESP-IDF is more verbose but useful for understanding the underlying peripheral.
