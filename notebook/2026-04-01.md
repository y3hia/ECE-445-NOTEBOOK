## MCU Constraints
Documented that while the ESP32-C3 (single core, 22 GPIOs) was sufficient for the breadboard demo, the final integrated system strictly requires the ESP32-S3. The dual core architecture and 45 GPIO pins are necessary to run the 6400 Hz stepper motor pulse generation on one core while executing the P&O MPPT algorithm and ADC polling on the second core without latency.

## Voltage Divider Calculations
Designed a custom voltage divider for the 23.9V panel to target a 2.5V maximum output, providing headroom for the ESP32's 3.3V ADC. Using a 10 kOhm base resistor (R2):

2.5V = 24V × (10,000 / (R1 + 10,000))
R1 = 86,000 Ω

We are utilizing 5% tolerance resistors to ensure the worst case scenario voltage remains safely below the MCU's 3.6V absolute maximum rating.

## Diagnostic Logic Validation
Finalized the dirt detection logic:
- The system establishes a unique baseline voltage for each panel during optimal lighting.
- If any panel deviates below its specific calibrated baseline by 25% while others remain nominal, the MCU flags it as dirty and deploys the gantry.
- This per-panel comparison is what allows the system to distinguish localized dirt from global cloud cover.

## Future Refinement (anticipated)
The 25% threshold may need to be tuned higher (50%) once we observe noise floor in real operation. Plan to expose this as a `#define` constant in the firmware so it's a one-line change later.
