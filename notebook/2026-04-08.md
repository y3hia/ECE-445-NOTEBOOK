# 2026-04-08

## Objective
Demonstration preparation and hardware troubleshooting.

## Record of what was done
- Realized the USB IC was missing from the board, preventing us from programming the MCU to drive the PWM signal directly.
- Decided to bypass the MCU for the demo and use a bench function generator to supply the PWM for the MPPT circuit instead.
- Discovered we lacked the necessary 3.3V to 5V level shifters. As a result, the ESP32 dev board could not properly drive the 5V inputs on the stepper motor drivers.
- Discussed writing the MPPT code in the meantime to demonstrate software progress despite the hardware integration delays.

## Workarounds for Demo
- Function generator on the gate of the buck MOSFET (since MCU can't drive PWM yet).
- Function generator on the stepper drivers' PUL+ pin (since level shifters not present).
- ESP32-C3 Super Mini for breadboard sanity checks of the firmware logic.

## Code Architecture Decisions
While the hardware is recovering, started laying out the firmware architecture:
- A non-blocking main loop that ticks MPPT every 200 ms.
- A motor state machine (IDLE / MOVING / DONE) so movements don't hang the loop.
- Per-panel detection state with debouncing so noise doesn't trigger false cleans.
- Manual override hooks via serial input.
