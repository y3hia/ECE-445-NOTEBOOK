# 2026-04-14

## Objective
Test buck converter design on isolated PCB.

## Record of what was done
- Had to physically cut a section of the PCB to allow the USB cable to plug into the receptacle properly.
- Brought up the buck converter on an extra PCB. On this board, only the buck components were soldered so we could test it in isolation using an external power supply and signal generator.

## Test 1
### Parameters
- Rout = 8 ohms
- Switching signal: 3.3V p-p, f = 100 kHz, 50% duty cycle

### Results
- Vin = 10 V, Iin = 0.41 A (initially seemed normal)
- Vout unexpectedly spiked from 4.5 V to 8 V
- The MOSFET switch started smoking. We immediately shut off the bench power.

### Analysis
We determined that the MOSFET switch failed because it exceeded its current ratings. With Rout = 8 ohms and Vout climbing to 8 V, the current draw exceeded the switch rating.

## Test 2
### Parameters
- Increased Rout to 15 ohms
- Same 100 kHz, 50% duty cycle switching signal

### Results
- Vin = 12 V, Iin = 0.31 A
- Vout = 6.536 V, Iout = 0.47 A

### Conclusion
The switch did not fail under these conditions, confirming that the converter design works at lower current.

## Lessons Learned
- Need to enforce current limiting in software once the MCU drives the gate, so the algorithm never picks a duty cycle that drives Iout past the switch rating.
- The MAX_DUTY constant in the MPPT code will be capped at a value that keeps the converter safely below the smoke-emitting threshold.
